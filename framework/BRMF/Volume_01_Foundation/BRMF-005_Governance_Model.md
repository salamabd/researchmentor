# BRMF-005 — Governance Model

Status: Draft 1.0
Classification: Constitutional Foundation

# 1. Purpose

This document defines the governance model of the Business Research Mentor (BRM)
project. It establishes how educational, architectural, and implementation
decisions are governed and how the framework evolves over time.

# 2. Governance Hierarchy

1. BRMF (Constitution)
2. Educational Decision Records (EDRs)
3. Architectural Decision Records (ADRs)
4. Specifications
5. Implementation
6. Validation

Higher levels govern lower levels.

# 3. Constitutional Authority

The Business Research Mentor Foundation (BRMF) is the highest authority for the
platform. Any lower-level document that conflicts with BRMF must be revised or
formally challenge BRMF through the governance process.

# 4. Stable vs Evolving Knowledge

## Stable
- Educational purpose
- Student-centred development
- Supervisor partnership
- Academic integrity
- Explainability
- Adaptive mentoring

## Evolving
- Research reasoning models
- Mentoring models
- Student state models
- Research constructs
- AI orchestration
- User interfaces

# 5. Decision Records

## Educational Decision Records (EDRs)

Capture educational theories, mentoring models, learning principles and their
rationale.

## Architectural Decision Records (ADRs)

Capture software architecture and engineering decisions.

# 6. Change Management

Changes to constitutional principles require:
- documented rationale
- impact assessment
- version increment
- approval

# 7. Traceability

Every specification should trace back to:
BRMF → EDR/ADR → Specification → Implementation → Validation.

# 8. Acceptance

This document becomes authoritative only after formal review and approval.
